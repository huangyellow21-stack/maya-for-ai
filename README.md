# AI 小助手

AI 小助手是一款用于 Autodesk Maya 的自然语言辅助插件。  
你可以直接用中文描述想做的事，例如创建物体、整理场景、查询信息、生成简单动画，插件会优先调用内置工具完成操作；当任务较复杂时，会先规划步骤，再在需要时执行。

---

## 它能做什么

AI 小助手主要帮助你完成以下几类工作：

- **场景查询**
  - 查看当前选择了什么
  - 概览场景中的模型、摄像机、灯光与动画对象
  - 根据上下文理解“它”“这些物体”等指代

- **建模与场景整理**
  - 创建常见基础物体，如立方体、球体、圆柱、平面
  - 复制、分组、冻结变换、居中轴心
  - 批量重命名或整理场景结构

- **动画辅助**
  - 创建基础位移、旋转、循环、弹跳等动画
  - 对已有关键帧进行调整
  - 结合当前对象与上下文生成可执行步骤

- **灯光与摄像机**
  - 创建摄像机
  - 快速布置三点光
  - 生成转台展示相关内容
  - 让摄像机看向目标或框选对象

- **复杂任务规划**
  - 对多步骤任务先进行分析和规划
  - 在执行前给出计划卡或确认提示
  - 必要时才生成 Python 执行代码，而不是默认直接写代码

---

## 适合的使用场景

你可以把它当作 Maya 中的一个辅助操作入口，适合以下场景：

- 想快速完成重复性操作  
  例如：批量创建物体、统一整理对象、批量赋材质、批量调整动画参数

- 想减少菜单和命令查找时间  
  例如：直接输入“创建一个摄像机并看向当前选择物体”

- 想在当前场景上下文里继续操作  
  例如：先创建一个球，再说“把它放到原点”“给它做一个弹跳”

- 想先看执行计划，再决定是否执行  
  例如：涉及多个步骤、多个对象或可能影响较大的操作

---

## 工作方式

AI 小助手并不是简单地把你的输入转成一段脚本。它的处理逻辑大致如下：

1. **理解你的请求**
2. **优先匹配现有 Maya 内置工具**
3. **对于复杂任务，先进行规划**
4. **必要时才生成 Python 执行**
5. **对高风险或不明确操作要求确认**

这意味着它会尽量使用更稳定、可控的方式完成操作，而不是一开始就直接执行自由脚本。

## 安装方式

### 方法一：拖拽安装（推荐）

1. 下载并解压项目文件
2. 确保目录结构完整，不要只复制其中一部分
3. 将根目录下的 `ins_aiformaya.py` 直接拖入 Maya 窗口
4. 按提示完成安装

安装完成后，插件会被复制到 Maya 的模块目录中，并写入对应的 `.mod` 文件。

> 提示：为保证稳定性，安装后建议重启一次 Maya。

---

## 启动方式

安装完成后，可在 Maya Script Editor 中执行：

```python
import aiformaya
aiformaya.show()
```

执行后会打开 AI 小助手 面板。

### 首次使用

**1. 配置 API Key**
打开面板后，先填写你使用的大模型 API Key。
当前配置界面支持切换不同 Provider，例如：
- DeepSeek
- Gemini

**2. 启动 Gateway**
点击面板中的 Start Gateway 启动本地网关服务。
当状态显示为 Connected 时，表示插件已经可以正常通信。

**3. 输入你的需求**
你可以直接输入中文，例如：
- 创建一个立方体
- 给当前物体做一个循环旋转动画
- 看看场景里有哪些摄像机
- 把这些物体打组并居中轴心
- 给当前选中的模型做三点布光

## 安全与确认机制

AI 小助手默认以“尽量可控”为原则工作。

**1. 优先使用已注册工具**
如果已经存在对应工具，会优先调用工具，而不是随意生成脚本。

**2. 复杂任务先规划**
当任务包含多个步骤或存在歧义时，系统会先整理执行计划，再决定下一步。

**3. 高风险操作需要确认**
对于删除、批量改动、可能影响较大的操作，系统会要求确认，而不会直接执行。

**4. 支持撤销**
多数场景操作可通过 Maya 常规撤销方式回退。
如果某些操作无法完全撤销，应以 Maya 当前场景状态和具体工具实现为准。

> 说明：AI 小助手可以提升操作效率，但不应替代你对场景结果的最终确认。对于重要场景，仍建议先保存文件或使用副本测试。

## 使用建议

为了获得更稳定的结果，建议这样描述需求：

**尽量明确对象与动作**
比如“给当前选中的球体做 1 到 30 帧的旋转动画”
比“让它动起来”更容易得到稳定结果

**复杂任务尽量分步骤**
比如先“创建三个球”
再“把它们沿 X 轴排开”
再“给它们分别添加旋转动画”

**涉及范围较大的操作时先确认**
比如删除、批量修改、场景清理

## 常见问题

**1. 为什么点击后没有响应？**
请先确认：
- API Key 已正确填写
- Gateway 已启动
- 面板状态为 Connected

**2. 为什么提示无法连接网关？**
通常是以下原因之一：
- 本地网关尚未启动
- 端口被占用
- Python 环境不可用
- 网络不可用
- 插件目录不完整，缺少 bridge 相关文件

**3. 为什么 AI 没有直接执行，而是先给计划？**
这通常说明任务较复杂，或者操作可能影响较大。
在这种情况下，先确认计划会更安全。

**4. 为什么没有按我预期执行？**
自然语言请求本身可能存在歧义。
建议把对象、范围、帧数、数量、目标说得更明确一些。

## 开发者说明

如果你希望扩展 AI 小助手，可以从以下部分入手：

**1. 新增工具**
在 `tools/` 中增加新的 Maya 工具，并注册到工具系统。
只要工具 Schema 正确，代理层就可以在合适的任务中调用它。

**2. 调整代理策略**
核心逻辑位于 `core/`，包括：
- 请求路由
- 意图解析
- 场景上下文
- 任务规划
- 执行与确认流程

**3. 手动调试网关**
如需单独调试本地网关，可在 `bridge/` 目录下手动启动服务。

示例：

```powershell
cd bridge
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt

$env:DEEPSEEK_API_KEY = "your_api_key"
python -m uvicorn server:app --host 127.0.0.1 --port 8765
```

## 说明

AI 小助手的目标是帮助 Maya 用户更自然地完成常见操作、减少重复劳动，并在复杂任务中提供更清晰的执行路径。
它并不保证对所有自然语言请求都一次性得到完全正确的结果，但会尽量在“可执行性、可理解性和安全性”之间保持平衡。
